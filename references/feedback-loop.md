# 反馈闭环参考

## 核心原则

反馈闭环不绑定任何特定的记忆工具或存储后端。Agent 在运行时自适应地使用当前环境可用的工具来存储和召回反馈。

## 自适应存储策略

| 优先级 | 存储方式 | 说明 |
|--------|---------|------|
| 1 | 记忆系统的存储功能 | Agent 的高阶记忆工具（如有） |
| 2 | Agent 原生记忆工具 | Agent 内置的记忆/持久化功能 |
| 3 | 本地缓存文件 | 快照文件 `interest-snapshot.json` |

## 用户反馈触发词

### confirm
准 / 确实 / 对 / 是的 / 就是这个 / 说得对 / 看了 / 不错 / 这个有用 / 有帮助

### dismiss
不准 / 不相关 / 不对 / 不是这样 / 别推这个 / 跟我没关系 / 我不关心这个 / 判断错了

### 不触发
嗯 / 好 / 行 / 知道了（模糊回应）

## 通用存储格式

无论用哪种存储工具，内容格式统一：

```
兴趣雷达反馈: [confirm|dismiss] — "内容标题" (score=82)
```

### 用记忆系统存储时
附带 context 和 tags 便于后续召回：
```
context: interest_radar_feedback
tags: [interest-radar, feedback, confirm|dismiss]
```

### 回退到本地文件时
在快照的 feedback 字段中追加：
```json
{
  "feedback": [
    {"type": "dismiss", "title": "文章标题", "score": 5, "time": 1715760000}
  ]
}
```

## 反馈影响

| 场景 | 影响 |
|------|------|
| 单条 confirm | 同类内容评分更自信，置信度可提一档 |
| 单条 dismiss | 同类内容降分 10-20 分 |
| 多次 dismiss（≥ 3 次） | 同类内容降入 ignore 档位 |
| 多次 confirm（≥ 3 次） | 同类内容上调 5-10 分 |

Agent 通过上下文召回自动带回反馈记录，在评分时自然调整。

## 边界

- ❌ 不绑定特定记忆工具
- ❌ 不修改评分算法代码
- ❌ 不要求用户反馈（只对明确信号反应）
- ✅ 隐式生效
