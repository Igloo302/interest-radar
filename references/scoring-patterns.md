# 评分模式参考

## 两种评分模式

### 模式 A：Agent 推理评分（推荐）

适用于 items ≤ 10 的场景。Agent 直接基于推理做评分，不依赖硬编码规则。

**流程：**
1. 读取每条内容的 title、content、source
2. 结合上下文召回结果（快照/记忆/项目笔记）
3. 按 SKILL.md Step 4 的 6 个维度逐条推理评分
4. 输出 score (0-100) + why_it_matters

**优点：** 评分质量高，能捕捉语义关联
**缺点：** 消耗 tokens，items 多时慢

### 模式 B：关键词快速初筛（辅助）

适用于 items > 10 的场景。先快速初筛，top_k × 3 条进入完整推理评分。

**流程：**
1. 从上下文召回中提取关键词列表（项目名、技术栈、兴趣标签）
2. 对每条 item 的 title 做关键词匹配
3. 按匹配数量 + 关键词权重排序
4. 取 top_k × 3 条进入模式 A

**注意：** 关键词匹配只是初筛，最终评分必须用模式 A 完成。

### 模式选择

| 场景 | 推荐模式 |
|------|---------|
| items ≤ 10 | 模式 A（直接推理） |
| items 10-15 | 模式 A（直接推理）— 12 条以内全量推理依然高效，无需初筛 |
| items 16-30 | 模式 B 初筛 → 模式 A 完整评分 |
| items > 30 | 模式 B 初筛 → 模式 A 评分 top_k × 3 条 |

## 评分维度详解

### 1. 项目影响（权重 35%）

判断内容是否直接影响 active project：

- **直接影响**（30-35分）：内容涉及的项目/技术栈与 active project 完全匹配
  - 例：Hermes Agent 项目 → Codex/MCP/Agent 框架更新
  - 例：html-anything 项目 → 编辑器/Agent 集成相关
- **间接影响**（15-29分）：内容涉及的技术方向与项目相关
  - 例：Agent 平台项目 → 通用 Agent 设计模式讨论
- **弱影响**（1-14分）：内容与项目有微弱关联
- **无影响**（0分）：完全不相关

### 2. 兴趣匹配（权重 20%）

判断内容是否匹配用户的 durable interest：

- **强匹配**（16-20分）：直接命中核心兴趣（XR、AI Agents、Personal Context）
- **中匹配**（8-15分）：命中边缘兴趣或相关领域
- **弱匹配**（1-7分）：泛泛相关
- **不匹配**（0分）

### 3. 变化幅度（权重 15%）

判断内容的事件类型：

- **重大变化**（13-15分）：release / breaking change / SDK update / paper
- **中等变化**（7-12分）：观点讨论 / 产品更新 / 新功能
- **小变化**（1-6分）：日常推文 / 转发 / 评论
- **无变化**（0分）：重复内容 / 旧闻

### 4. 可行动性（权重 15%）

判断内容是否能触发具体行动：

- **高可行动**（13-15分）：需要更新代码/配置/PRD
- **中可行动**（7-12分）：需要阅读/了解/评估
- **低可行动**（1-6分）：纯信息获取
- **不可行动**（0分）：纯娱乐/八卦

### 5. 来源可信度（权重 10%）

- **官方**（9-10分）：项目官方账号/文档
- **一手**（6-8分）：作者本人/核心贡献者
- **二手**（3-5分）：媒体报道/转发
- **不可信**（0-2分）：未知来源

### 6. 时效性（权重 5%）

- **24h 内**（5分）
- **一周内**（3-4分）
- **更早**（1-2分）
- **旧闻**（0分）

## 分桶规则

| 分数 | 档位 | 含义 |
|------|------|------|
| 80-100 | 🔥 top | 强烈建议阅读 |
| 60-79 | 👁 watch | 值得关注 |
| 30-59 | 🔇 silent | 有关联但不需要立即行动 |
| 0-29 | ⏭ ignore | 不相关 |

### Top 档位二次验证（评分 ≥ 80 时）

高相关度内容必须执行语义验证，防止过度匹配：

1. 从实体中提取与评分强相关的关键词（最多 3 个）
2. 验证这些关键词在内容中的上下文是否确实与项目/兴趣匹配
3. 例：内容中提到 "agent" → 确认是否真的是 AI Agent 而非 "travel agent"
4. 验证不通过时，下调至 watch 档位，置信度降一档

此步骤只做语义验证，不做额外召回，避免无限循环。
