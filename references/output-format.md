# 兴趣雷达输出格式参考

## 微信端输出模板

```
📡 兴趣雷达

🔗 [内容标题]
来源：[来源域名]

📊 相关度：[score]/100（[bucket中文]）
🎯 关联：[项目名/兴趣名]

💡 为什么相关：
[why_it_matters — 1-2句话，必须关联到具体的项目/兴趣/决策，指出具体影响点]

📋 建议行动：[read/watch/save/ignore]
   具体建议：[如"更新 PRD 技术风险部分"]

📎 判断依据：
• [依据1：来自Hindsight/Memory/Obsidian的具体内容]
• [依据2]

置信度：High/Medium/Low
```

## 分桶中文对照

| 英文 | 中文 | 含义 |
|------|------|------|
| top | 🔥 强烈建议阅读 | 80-100分 |
| watch | 👀 值得关注 | 60-79分 |
| silent | 💤 有关联但不需要立即行动 | 30-59分 |
| ignore | ❌ 不相关 | 0-29分 |

## 置信度说明

- **High**：多源召回结果一致，上下文匹配度高
- **Medium**：单源召回，或上下文匹配度一般
- **Low**：内容过短/信息不足，或回退到种子兴趣

## 注意事项

- 微信不支持 markdown 表格，用列表格式
- why_it_matters 必须具体，避免"因为你对 XX 感兴趣"这种废话
- 判断依据必须列出具体的上下文来源（Hindsight/Memory/Obsidian）
