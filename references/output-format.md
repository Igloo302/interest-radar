# 兴趣雷达输出格式参考

## 标准输出模板

```
📡 兴趣雷达

🔗 [内容标题]
来源：[来源域名]

📊 相关度：[score]/100（[bucket中文]）
🎯 关联：[项目名/兴趣名]

💡 为什么相关：
[why_it_matters — 1-2句话，必须关联到具体的项目/兴趣/决策，指出具体影响点]

📋 建议行动：[read/watch/save/ignore]
   具体建议：[如"更新 PRD 技术风险部分"]

📎 判断依据：
• [依据1：来自上下文的具体内容]
• [依据2]

置信度：High/Medium/Low
```

## batch_judge 汇总模板

```
📡 兴趣雷达 · 批量过滤

📊 汇总：[总数] 条候选 → [推送数] 条推送 🔥 | [观察数] 条观察 👀 | [归档数] 条归档 💤

🔥 推送：
1. [标题] [分数]/100 — [原因]

👀 观察：
2. [标题] [分数]/100 — [原因]

💤 归档（[数量] 条略）

判断依据：[上下文来源]
```

## 分桶中文对照

| 英文 | 中文 | 分数 |
|------|------|------|
| top | 🔥 强烈建议阅读 | 80-100 |
| watch | 👀 值得关注 | 60-79 |
| silent | 💤 有关联但不需要立即行动 | 30-59 |
| ignore | ❌ 不相关 | 0-29 |

## 置信度说明

- High：多源召回结果一致，上下文匹配度高
- Medium：单源召回，或上下文匹配度一般
- Low：内容过短/信息不足，或回退到种子兴趣

## 平台适配

不同平台需调整输出格式：
- 微信：不支持 markdown 表格，用列表格式
- Telegram：支持 markdown
- 飞书：支持富文本和卡片
- CLI：纯文本即可
